whilefor Encontrar el error — for vs while
Codigo con trampa — este programa nunca termina
Dificil

Este codigo tiene un error logico grave. El programador quiso hacer un menu que se muestre mientras el usuario no elija "3". Pero hay algo mal que hace que el programa nunca pueda terminar. Encontralo y propone la correccion.

opcion = "0" for i in range(10): # ERROR ACA print("1. Ver temperatura") print("2. Ver humedad") print("3. Salir") opcion = input("Opcion: ") if opcion == "1": print("Temperatura: 22.5 C") elif opcion == "2": print("Humedad: 58%") elif opcion == "3": print("Saliendo...")
Analizá el error en el papel, luego escribi tu respuesta

Que hace mal? Cual es la consecuencia? Como se corrige?


El ERROR es que uso "for i in range(10)" en lugar de "while True". Consecuencias del error: - El for hace exactamente 10 repeticiones, sin importar lo que el usuario elija. - Si el usuario elige "3", imprime "Saliendo..." pero el for SIGUE igual. - No hay forma de salir antes de las 10 vueltas. CORRECCION: reemplazar el for por while True + break: while True: # en lugar de for i in range(10) print("1. Ver temperatura") ... elif opcion == "3": print("Saliendo...") break # agregar el break para salir El menu no tiene un numero fijo de repeticiones. Un "for" se usa cuando sabes CUANTAS veces repetir. Un "while" se usa cuando dependes de una CONDICION.
